Поддержка беспроводных подключений

Режим беспроводной сети

Режимом беспроводной сети называется настройка стандарта IEEE 802.11, который будет использоваться сетью. Существует четыре версии стандарта IEEE 802.11, которые описывают различные характеристики обмена данными по беспроводной сети — 802.11a, 802.11b, 802.11g и 802.11n. На рис. 1 представлены дополнительные сведения о каждом стандарте.

Большинство интегрированных маршрутизаторов поддерживают стандарты 802.11b, 802.11g и 802.11n. Данные три технологии совместимы между собой, однако все устройства в сети должны использовать один общий стандарт. Например, если маршрутизатор 802.11n подключён к ноутбуку с использованием стандарта 802.11n, вся сеть использует стандарт 802.11n. Теперь попробуем подключить к сети беспроводной принтер стандарта 802.11b. И маршрутизатор, и ноутбук снова будут во всех случаях использовать для обмена данными более медленный стандарт 802.11b. Таким образом, если к сети подключены беспроводные устройства, использующие более раннюю версию стандарта, быстродействие всей сети снизится. Этот факт необходимо учитывать при принятии решения относительно сохранения в сети беспроводных устройств, использующих более ранние версии стандарта.

Service Set Identifier (SSID) (Имя сети)

В радиусе действия может быть обнаружено множество беспроводных сетей. Важно проследить, чтобы беспроводные устройства были подключены к правильной беспроводной глобальной сети WLAN. Для этого используется идентификатор набора услуг (SSID).

SSID представляет собой чувствительное к регистру буквенно-числовое имя домашней беспроводной сети. Имя может содержать не более 32 символов. Идентификатор SSID сообщает беспроводным устройствам, к какой беспроводной сети WLAN они принадлежат и с какими устройствами они взаимодействуют. Для обеспечения связи все беспроводные устройства в сети WLAN должны иметь общий идентификатор SSID, независимо от типа установки сети WLAN.

Беспроводной канал

Каналы создаются путём деления доступного радиочастотного спектра. Каждый канал может использоваться в качестве несущей для другого сеанса связи. Такой процесс можно сравнить с передачей нескольких телевизионных каналов по одной среде. Несколько точек доступа могут работать в непосредственной близости одна к другой, если они используют разные каналы связи.
