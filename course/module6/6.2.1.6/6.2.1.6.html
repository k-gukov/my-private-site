Пример таблицы маршрутизации узла IPv6

Выходные данные в таблице маршрутизации IPv6 отличаются заголовками столбцов и форматом из-за более длинных адресов IPv6.

Раздел таблицы маршрутизации IPv6 отображает четыре столбца, которые содержат следующую информацию.

  • If (Если): содержит номера интерфейсов из раздела «Список интерфейсов» команды netstat –r. Номера интерфейсов соответствуют интерфейсу с поддержкой сети на узле, включая адаптеры Ethernet, Wi-Fi и Bluetooth.

  • Metric (Метрика): содержит стоимость каждого маршрута к адресату. Меньшие цифры означают предпочтительные маршруты.

  • Network Destination (Сеть назначения): список досягаемых сетей.

  • Gateway (Шлюз): содержит адрес, который используется локальным узлом для передачи пакетов удалённому сетевому адресату. Сообщение «On-link» означает, что узел в настоящее время подключён к шлюзу.

Например, на рисунке показан раздел с маршрутом IPv6, который сгенерирован с использованием команды netstat –r для определения следующих сетевых адресатов.

  • ::/0 — IPv6-эквивалент для локального маршрута по умолчанию.

  • ::1/128 — эквивалент loopback-адреса IPv4, который предоставляет сервисы локальному узлу.

  • 2001::/32 — глобальный индивидуальный сетевой префикс.

  • 2001:0:9d38:953c:2c30:3071:e718:a926/128 — глобальный индивидуальный IPv6-адрес локального компьютера.

  • fe80::/64 — адрес маршрута сети локального канала, который представляет все компьютеры в сети IPv6 локального канала.

  • fe80::2c30:3071:e718:a926/128 — IPv6-адрес локального канала локального компьютера.

  • ff00::/8 — специальные зарезервированные широковещательные адреса класса D, которые соответствуют адресам IPv4 224.x.x.x.

Примечание. Интерфейсы протокола IPv6 обычно содержат два IPv6-адреса: адрес локального канала и глобальный индивидуальный адрес. Кроме того, обратите внимание, что в IPv6 отсутствуют широковещательные адреса. Адреса IPv6 будут рассматриваться далее в следующей главе.
