Использование команды «show ip interface brief»

Аналогично тому, как эти команды и программы используются для проверки конфигурации узла, их можно использовать для проверки интерфейсов промежуточных устройств. Cisco IOS предоставляет команды для проверки работоспособности интерфейсов маршрутизатора и коммутатора.

Проверка интерфейсов маршрутизатора

К наиболее часто используемым командам относится команда show ip interface brief. Эта команда выводит выходные данные в более компактном формате, чем команда show ip interface. Она предоставляет сводку по ключевым данным для всех сетевых интерфейсов на маршрутизаторе.

На рис. 1 показана топология, используемая в данном примере.

На рис. 2 нажмите кнопку «R1». В выходных данных команды show ip interface brief отображаются все интерфейсы на маршрутизаторе, IP-адрес, назначенный для каждого из интерфейсов (если есть), и рабочее состояние интерфейса.

В соответствии с выходными данными интерфейс FastEthernet 0/0 имеет IP-адрес 192.168.254.254. Последние два столбца в этой строке отображают статусы 1-го и 2-го уровней данного интерфейса. Статус up в столбце «Статус» указывает на то, что этот интерфейс работает на уровне 1. Статус up в столбце «Протокол» указывает на то, что работает протокол 2-го уровня.

Обратите также внимание, что интерфейс Serial 0/0/1 неактивен. На это указывает статус administratively down в столбце «Статус».

Как и в случае с любым оконечным устройством, для проверки работоспособности 3-го уровня можно использовать команды ping и traceroute. В данном примере обе команды ping и trace показывают успешное подключение.

Проверка интерфейсов коммутаторов

На рис. 2 нажмите кнопку «S1». Команду show ip interface brief также можно использовать для проверки статуса интерфейсов коммутатора. IP-адрес коммутатора применяется к интерфейсу сети VLAN. В этом случае интерфейсу Vlan1 назначается IP-адрес 192.168.254.250, интерфейс активен и работает.

В этих выходных данных также указано, что интерфейс FastEthernet0/1 отключён. Это означает, что к интерфейсу не подключены устройства, или сетевой интерфейс устройства, подключённое к этому интерфейсу, не работает.

И, наоборот, в выходных данных указано, что интерфейсы FastEthernet0/2 и FastEthernet0/3 работают. Это обозначено статусом up в столбцах «Статус» и «Протокол».

Коммутатор также может проверить подключение на 3-м уровне с помощью команд show ip interface brief и traceroute. В данном примере обе команды ping и trace показывают успешное подключение.

Важно учитывать, что коммутатору необязательно требуется IP-адрес для пересылки кадров на 2-м уровне. IP-адрес требуется только в том случае, если управление коммутатором осуществляется по сети посредством Telnet или SSH. Если сетевой администратор планирует удалённо подключаться к коммутатору из расположения за пределами локальной сети LAN, необходимо также настроить шлюз по умолчанию.
