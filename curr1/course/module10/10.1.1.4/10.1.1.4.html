Протоколы уровня приложений TCP/IP

Хотя в модели OSI функции уровня приложений, уровня представления и сеансового уровня отделены друг от друга, наиболее известные и широко используемые приложения TCP/IP объединяют в себе функции всех трёх уровней.

Протоколы прикладного уровня TCP/IP определяют форматы и управляют данными, необходимыми для многих распространённых функций обмена данными через Интернет. Ниже перечислены некоторые из таких протоколов TCP/IP.

  • Служба доменных имён (DNS): используется для преобразования имён интернет-ресурсов в IP-адреса.

  • Telnet: используется для предоставления удалённого доступа к серверам и сетевым устройствам.

  • Простой протокол передачи электронной почты (SMTP): используется для передачи почтовых сообщений и вложений.

  • Протокол динамической конфигурации сетевого узла (DHCP): используется для назначения узлу IP-адреса, маски подсети, шлюза по умолчанию и адресов DNS-серверов.

  • Протокол передачи гипертекста (HTTP): используется для передачи веб-страниц из Интернета.

  • Протокол передачи файлов(FTP): используется для передачи файлов между системами.

  • Простой протокол передачи файлов (TFTP): используется для передачи файлов без установления соединения.

  • Протокол BOOTP (предшественник протокола DHCP). Используется для получения данных об IP-адресе во время загрузки.

  • Почтовый протокол (POP): используется почтовыми клиентами для получения электронной почты с удалённых серверов.

  • Протокол доступа к сообщениям в Интернете (IMAP): используется для клиентского подключения к серверу электронной почты и просмотра почтовых сообщений.

Протоколы уровня приложений используются исходными и конечными устройствами во время сеанса связи. Для успешного обмена данными протоколы уровня приложений на исходном и конечном узлах должны быть совместимыми.
